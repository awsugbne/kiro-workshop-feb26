Instead of building everything from scratch, leverage community resources to accelerate your development. **Promptz.dev** is a community library for sharing Kiro resources.

## What is Promptz.dev?

[Promptz.dev](https://www.promptz.dev) is a community-driven library where developers share:

<CardGrid>
  <Card title="Powers" icon="rocket">
    Pre-built Power packages like Draw.io diagrams, PowerPoint slides, and more.
  </Card>
  <Card title="Agents" icon="puzzle">
    Pre-configured agents for specific tasks like the Bedrock Model Agent.
  </Card>
  <Card title="Steering Docs" icon="document">
    Ready-to-use coding standards, style guides, and project conventions.
  </Card>
  <Card title="Prompts & Hooks" icon="pencil">
    Effective prompts and automation hooks tested by the community.
  </Card>
</CardGrid>

## Finding Resources on Promptz

<Steps>

1. **Browse the catalog**

   Visit [promptz.dev](https://www.promptz.dev) and browse by category (Powers, Agents, Steering, Prompts).

2. **Filter by compatibility**

   Resources are tagged for IDE and/or CLI compatibility. Filter based on your workflow.

3. **Review and install**

   Each resource includes documentation and installation instructions.

4. **Contribute back**

   Share your own resources to help the community!

</Steps>

## Practical Examples

### Finding Steering Docs

Instead of writing PEP 8 or accessibility guidelines from scratch:

1. Search Promptz for "PEP 8" or "accessibility"
2. Find community-tested steering docs
3. Download and add to your `.kiro/steering/` directory
4. Customize as needed for your project

### Using Community Prompts

Find effective prompts for common tasks:

- **Code review prompts** — Thorough review checklists
- **Refactoring prompts** — Safe refactoring patterns
- **Documentation prompts** — Consistent doc generation
- **Testing prompts** — Comprehensive test generation

<Aside type="tip">
Community prompts are often more refined than first attempts. They've been tested and improved by multiple developers.
</Aside>

## Contributing to the Community

Share your own resources:

<Steps>

1. **Create your resource**

   Build a steering doc, prompt, hook, or Power that solves a real problem.

2. **Document it well**

   Include clear descriptions, usage examples, and any prerequisites.

3. **Submit to Promptz**

   Follow the contribution guidelines on Promptz.dev.

4. **Get feedback**

   The community will test and suggest improvements.

</Steps>

<Aside type="note">
Contributing back helps the ecosystem grow. Even small improvements to existing resources are valuable!
</Aside>

## Recommended Starting Points

If you're new to community resources, start with these:

1. **Browse steering docs** — Find coding standards for your language/framework
2. **Install a popular Power** — Try Draw.io or another highly-rated Power
3. **Join Discord** — Introduce yourself and ask questions
4. **Star useful repos** — Build your collection of go-to resources
